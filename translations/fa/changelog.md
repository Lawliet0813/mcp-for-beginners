# تغییرات: برنامه درسی MCP برای مبتدیان

این سند به عنوان یک رکورد از تمام تغییرات مهمی که در برنامه درسی پروتکل مدل کانتکست (MCP) برای مبتدیان انجام شده است، عمل می‌کند. تغییرات به ترتیب زمانی معکوس (جدیدترین تغییرات در ابتدا) ثبت شده‌اند.

## ۱۵ سپتامبر ۲۰۲۵

### گسترش موضوعات پیشرفته - حمل‌ونقل‌های سفارشی و مهندسی کانتکست

#### حمل‌ونقل‌های سفارشی MCP (05-AdvancedTopics/mcp-transport/) - راهنمای پیاده‌سازی پیشرفته جدید
- **README.md**: راهنمای کامل پیاده‌سازی برای مکانیزم‌های حمل‌ونقل سفارشی MCP
  - **حمل‌ونقل Azure Event Grid**: پیاده‌سازی جامع بدون سرور و مبتنی بر رویداد
    - مثال‌هایی با C#، TypeScript و Python همراه با ادغام Azure Functions
    - الگوهای معماری مبتنی بر رویداد برای راه‌حل‌های مقیاس‌پذیر MCP
    - دریافت‌کننده‌های Webhook و مدیریت پیام‌های مبتنی بر فشار
  - **حمل‌ونقل Azure Event Hubs**: پیاده‌سازی حمل‌ونقل با ظرفیت بالا برای سناریوهای کم‌تاخیر
    - قابلیت‌های جریان بلادرنگ برای سناریوهای کم‌تاخیر
    - استراتژی‌های تقسیم‌بندی و مدیریت نقاط بررسی
    - دسته‌بندی پیام‌ها و بهینه‌سازی عملکرد
  - **الگوهای یکپارچه‌سازی سازمانی**: مثال‌های معماری آماده تولید
    - پردازش توزیع‌شده MCP در چندین Azure Functions
    - معماری‌های حمل‌ونقل ترکیبی با ترکیب چندین نوع حمل‌ونقل
    - استراتژی‌های دوام پیام، قابلیت اطمینان و مدیریت خطا
  - **امنیت و نظارت**: ادغام Azure Key Vault و الگوهای مشاهده‌پذیری
    - احراز هویت با هویت مدیریت‌شده و دسترسی با کمترین امتیاز
    - تله‌متری Application Insights و نظارت بر عملکرد
    - قطع‌کننده‌های مدار و الگوهای تحمل خطا
  - **چارچوب‌های تست**: استراتژی‌های جامع تست برای حمل‌ونقل‌های سفارشی
    - تست واحد با دوبل‌های تست و چارچوب‌های شبیه‌سازی
    - تست یکپارچه‌سازی با Azure Test Containers
    - ملاحظات تست عملکرد و بار

#### مفاهیم اصلی (01-CoreConcepts/) - مدرن‌سازی کامل
- **به‌روزرسانی نسخه پروتکل**: به‌روزرسانی برای ارجاع به مشخصات فعلی MCP 2025-06-18 با نسخه‌بندی مبتنی بر تاریخ (YYYY-MM-DD)
- **تصفیه معماری**: توضیحات پیشرفته درباره میزبان‌ها، کلاینت‌ها و سرورها برای انعکاس الگوهای معماری فعلی MCP
  - میزبان‌ها اکنون به وضوح به عنوان برنامه‌های هوش مصنوعی که چندین اتصال کلاینت MCP را هماهنگ می‌کنند تعریف شده‌اند
  - کلاینت‌ها به عنوان اتصال‌دهنده‌های پروتکل که روابط یک‌به‌یک با سرورها را حفظ می‌کنند توصیف شده‌اند
  - سرورها با سناریوهای استقرار محلی در مقابل راه دور بهبود یافته‌اند
- **بازسازی ابتدایی‌ها**: بازسازی کامل ابتدایی‌های سرور و کلاینت
  - ابتدایی‌های سرور: منابع (منابع داده)، پرامپت‌ها (قالب‌ها)، ابزارها (توابع قابل اجرا) با توضیحات و مثال‌های دقیق
  - ابتدایی‌های کلاینت: نمونه‌گیری (تکمیل‌های LLM)، استخراج (ورودی کاربر)، ثبت (اشکال‌زدایی/نظارت)
  - به‌روزرسانی شده با الگوهای فعلی کشف (`*/list`)، بازیابی (`*/get`) و اجرا (`*/call`)
- **معماری پروتکل**: معرفی مدل معماری دو لایه
  - لایه داده: پایه JSON-RPC 2.0 با مدیریت چرخه عمر و ابتدایی‌ها
  - لایه حمل‌ونقل: STDIO (محلی) و HTTP قابل جریان با SSE (راه دور)
- **چارچوب امنیتی**: اصول جامع امنیتی شامل رضایت صریح کاربر، حفاظت از حریم خصوصی داده‌ها، ایمنی اجرای ابزار و امنیت لایه حمل‌ونقل
- **الگوهای ارتباطی**: به‌روزرسانی پیام‌های پروتکل برای نمایش جریان‌های اولیه‌سازی، کشف، اجرا و اعلان
- **مثال‌های کدنویسی**: تازه‌سازی مثال‌های چندزبانه (.NET، Java، Python، JavaScript) برای انعکاس الگوهای فعلی SDK MCP
